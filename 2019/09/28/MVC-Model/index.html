浅聊MVC模式

  2019-09-28


MVC即"Model"模型、"view"视图、"controller"控制的缩写。

这个模式将应用程序的结构分为三层：

★ Model

Model是整个应用程序的核心：也是处理操作数据和执行核心功能的主要部分

★ View

View是呈现用户输出结果的最外层，它作为视图层给用户提供操作与互动

★ Controller

Controller就是中间的控制层，通过连接View视图层与Model层，将视图层的操作指令发送至Model层从而对数据进行操作而产生相对应的结果，然后返回View层重新呈现。

★ 核心概念

MVC模式的核心在于:

Model必须不依赖于Controller层和View层，换而言之，Model, View, Controller必须相互独立

其关键的思想在于能创建模块化的代码从而能提高代码复用而不需要进行改动

★ Example

  1. 比如说你和朋友约在一个餐厅内享受晚餐:
  • 你可能会在餐桌上看菜单并点菜（view视图层)
  • 这时服务员(controller)并不知道你是谁，会点些啥，他（她）只负责帮你写下你的订单细节。
  • 但服务员并不帮你烹饪，他只是帮你将订单传递给厨师(Model)
  • 然后厨师(Model)会从冰箱里拿出相应的食材(Data)帮你进行烹饪。
  • 最后美味的食物终于完成，厨师将完成的食物交给服务员，这时服务员知道你点了些啥，然后知道应该用怎样服务方式来给你上菜。
  1. 现代汽车的架构其实也可以看成是MVC模型:
  • View = User Interface: (排档、刹车、油门、仪表盘、方向盘)
  • Controller = Mechanism: （变速转动装置，万向节）
  • Model = Engine(引擎)
  • Storage、Data = 油箱与汽油

这样的汽车结构就显得很清楚，你可以分别对不同的层进行各式各样的折腾和改造以及优化：比如对Engine部分进行改造、换更好的燃油让它马力更足，而不更换汽车的操作部分和外壳。

★ Summary

MVC模式的目的在于最大化的创建可复用代码, 它通过抽象(abstract)出Model, View, Controller三个独立的模块，这样的设计可以让Application应用允许扩展和升级而对数据模型有着最小的改动
